Overview of acquired skills

As part of the Bachelor's Degree in Computer Science, specialisation Application Development: Design, Development, Validation (RACDV), skill development is based on a national framework structured by levels. In semester 4, some skills reach level 2, corresponding to an intermediate degree of autonomy and proficiency, directly linked to professional scenarios.

This portfolio is structured around three level 2 skills, presented below, each accompanied by its critical learnings (CL) and associated essential components (EC).

Develop

Develop an application

CL1 Design and implement functional and non-functional specifications from requirements
CL2 Apply accessibility and ergonomics principles
CL3 Adopt good design and programming practices
CL4 Verify and validate application quality through testing

Optimize

Optimize IT applications

CL1 Choose complex data structures suited to the problem
CL2 Use suitable algorithmic techniques for complex problems (e.g., optimization, AI...)
CL3 Understand the issues and methods of data and code security
CL4 Assess the environmental and societal impact of proposed solutions

Verify

3. Verify and validate an application

CL1 Identify test types appropriate to a given situation
CL2 Create test sets and automate testing
CL3 Interpret results to validate or correct implementations
CL4 Document and formalize validation procedures

These skills form the foundation of the RACDV pathway, combining rigorous design, efficient programming, and methodical validation. They are central to the expectations of the Bachelor in Computer Science, and their progressive acquisition is a core aspect of the semester 4 evaluation.

SAE – Very Bad Split: Event Finance Management Web App

A full-stack event budgeting application developed in PHP (MVC-S), enhanced with a custom-built JavaScript reactivity library and a REST API. Users can create events, invite members, log expenses, and calculate debt balances automatically based on individual participation in shared expenses. The project originated from a flawed application with security holes, a single-table database, and no separation of concerns. We completely restructured and rebuilt the system.

Project Context: The original app had no architecture, was riddled with vulnerabilities, and used a database not even in First Normal Form. The challenge was to audit, refactor and secure the code, normalize the database step-by-step, and modernize the interface with reactivity and asynchronous API calls.

My Role & Responsibilities: I analyzed and documented security flaws (SQLi, XSS, weak authentication), redesigned and normalized the database schema into 3NF with justification, implemented the application’s core services and API endpoints in PHP, and developed the reactivity layer in JavaScript to make the UI fully interactive without page reloads. I also participated in the virtualisation of the project using Docker Compose.

Technologies Used: PHP (MVC-S), JavaScript, custom reactive library, MySQL, Docker, Docker Compose, REST API, Twig, AJAX.

Skills Developed:

  • Audited application quality and security, then proposed and applied structural improvements (AC3).
  • Reconstructed a database model by applying normalization (1NF → 3NF) with documented decompositions (AC1).
  • Implemented secure back-end architecture and REST API with JWT authentication (AC3, CE1.06).
  • Created a dynamic user interface with a custom reactive JavaScript layer and API consumption (AC2, AC4).
  • Applied the MVC-S architecture and introduced modular services and unit tests with PHPUnit (AC3, AC4).
  • Used Docker and Docker Compose for complete service virtualization and environment consistency (CE1.06).

Key Takeaways: This project was a deep dive into professional refactoring practices. It reinforced my skills in code quality, security, database modeling, API design, frontend/backend decoupling, and full-stack project deployment through containerization.

Skull – Online 3D Bluff Game

A browser-based 3D remake of the board game Skull, created just after my internship by reusing and expanding the technologies I had learned. The game emphasizes strategy and bluff mechanics, all within a modern and scalable architecture.

Project Context: This solo project was designed as a full 3D online multiplayer version of the game Skull, a strategic game of bluff and betting. The goal was to reproduce the gameplay faithfully while experimenting with modern frontend/backend architecture.

My Role & Responsibilities: I developed the entire frontend in React with Three.js for 3D rendering and styled it using Tailwind CSS. The back-end was initially prototyped in TypeScript, then completely reimplemented in Java using a hexagonal architecture to ensure loose coupling and testability. I designed the domain logic around SOLID principles to support scalability and code clarity.

Technologies Used: React, Three.js, Tailwind CSS, TypeScript, Java, Hexagonal architecture, WebSocket (for real-time communication).

Skills Developed:

  • Reused technologies from my internship in a new project context to reinforce my learning (AC1).
  • Applied modern frontend frameworks and 3D engines to create an interactive and immersive user experience (AC2, CE1.03).
  • Designed a modular back-end with clear domain logic using the hexagonal architecture and SOLID principles (AC3, CE1.06).
  • Validated the system through component testing and real-time gameplay debugging (AC4).

Key Takeaways: This project helped me solidify the architectural concepts discovered during my internship, deepen my mastery of 3D rendering for the web, and explore advanced back-end design patterns in Java.
